# 普适性 Bot 接入 — 群内交互式注册

在测试群里 @Inspector，无需提前手改 `bots.yaml`，即可完成「配置 → 校验 → 巡检」。

## 快速开始

```
@巡检助手 测试 你的Bot名称
```

1. Inspector 发送**配置卡片**（或文字模板）
2. 填写被测 Bot 的 `App ID`、`open_id`、能力标签等
3. 点击 **提交并校验**
4. 若失败：按卡片提示拉群、补 open_id 等，再点 **修改配置**
5. 全部通过后点击 **确认有效，开始 P0 巡检**

仅保存配置、暂不巡检：

```
@巡检助手 注册 你的Bot名称
```

## 自动填充

| 字段 | 来源 |
|------|------|
| `normal_group` | 当前触发指令的群 chat_id |
| 负责人 | 表单可选填写 |

## 校验项

- App ID / open_id 格式
- 被测 Bot 是否在测试群（需你将 Bot 与 Inspector 拉入同一群，并 @ 被测 Bot 一次以便解析 open_id）
- 私聊通道探测（向 open_id 发一条探测消息）
- 话题群（若勾选 `topic_reply`）
- 文档链接（若勾选 `doc_access`）
- 健康检查 URL（若填写）

## 配置落盘

动态注册的 Bot 写入 `config/bots_registered.yaml`，与手写的 `config/bots.yaml` 合并加载。

## 开放平台配置（Inspector 应用）

`card.action.trigger` **不是事件**，不要到「事件配置」里搜。它和 `im.message.receive_v1` 是两套订阅。

| 能力 | 所在页签 | 中文名称 | 技术标识 |
|------|----------|----------|----------|
| 收群里 @ 消息 | **事件配置** | 接收消息 | `im.message.receive_v1` |
| 配置卡片点「提交」 | **回调配置** | **卡片回传交互**（新版） | `card.action.trigger` |

### 正确操作步骤

1. 登录 [飞书开放平台](https://open.feishu.cn/app) → 进入 **Inspector（巡检助手）** 应用  
2. 左侧 **开发配置 → 事件与回调**  
3. 点顶部 **「回调配置」** 标签（不是「事件配置」）  
4. **订阅方式** 选 **「使用长连接接收回调」**  
   - 保存前须先在本机运行 `python -m src.chat_trigger` 并保持在线（与事件长连接相同）  
5. 在 **「已订阅的回调」** 区域点 **「添加回调」**  
6. 搜索 **「卡片回传交互」**（不要搜 `card.action.trigger`，事件列表里不会出现）  
7. 选择 **新版「卡片回传交互」**（`card.action.trigger`）  
   - 不要选带 **（旧）** 的 `card.action.trigger_v1`（旧版不支持长连接回调）  
8. **创建版本并发布** 到目标租户  

### 你截图里搜不到的原因

- 当前在 **「事件配置」→ 添加事件** 里搜索 → 这里只有事件，没有卡片回调  
- 切到 **「用户身份订阅」** 也无效 → 卡片回调在 **「回调配置」→ 应用身份** 下添加  

### 暂时不配回调也能用

若暂时无法添加回调，Inspector 仍会发**文字配置模板**。按模板回复 `app_id:` / `open_id:` 等行后，发送 **「提交配置」** 即可完成校验（只是不能点卡片按钮）。

## 文字兜底

若卡片未显示，按 Inspector 发出的模板回复 `key: value` 行，然后发送 `提交配置`。

取消当前流程：发送 `取消`。

## 已配置 Bot

若 Bot 已在台账且 `app_id`、`open_id`、`normal_group` 齐全，可直接：

```
@巡检助手 巡检 p0 你的Bot名称
```

或使用 `测试` 命令，将跳过注册直接进入巡检。
